# أوربت ERP — نظام إدارة الأعمال المتكامل

تطبيق ويب جاهز للتشغيل بدون الحاجة لـ Node.js. كل البيانات تُحفظ تلقائيًا على Firebase Firestore.

## التشغيل المحلي (بدون أي تثبيت)

افتح ملف `index.html` مباشرة في أي متصفح حديث (Chrome / Edge / Safari / Firefox).
ملاحظة: بعض المتصفحات تمنع Service Worker عند فتح الملفات بـ `file://` — للحصول على تجربة كاملة استخدم خادم محلي بسيط:

```
# لو عندك بايثون
python3 -m http.server 8080

# أو بأي خادم ساكن آخر، ثم افتح:
# http://localhost:8080
```

## النشر على GitHub Pages

1. أنشئ مستودع جديد على GitHub.
2. ارفع كل محتويات هذا المجلد (بما فيها `.nojekyll`) إلى الفرع `main` (أو `gh-pages`).
3. من إعدادات المستودع → Pages → Source: اختر الفرع والمجلد `/ (root)`.
4. التطبيق سيظهر على: `https://<username>.github.io/<repo>/`.

التطبيق يستخدم مسارات نسبية (`./`) لذا يعمل تلقائيًا تحت أي مسار فرعي.

## النشر على Netlify / Vercel / Cloudflare Pages

اسحب وأفلت المجلد كاملًا في لوحة النشر — سيشتغل فورًا.

## التثبيت كتطبيق على الهاتف

- **أندرويد (Chrome):** افتح الموقع → القائمة → "إضافة إلى الشاشة الرئيسية" أو "تثبيت التطبيق".
- **آيفون (Safari):** افتح الموقع → زر المشاركة → "إضافة إلى الشاشة الرئيسية".
- **ويندوز / ماك (Chrome / Edge):** أيقونة التثبيت تظهر في شريط العنوان.

سيظهر التطبيق مثل أي تطبيق أصلي بأيقونته الخاصة، ويعمل بدون شريط المتصفح.

## الحفظ السحابي

كل البيانات (المنتجات، الموظفون، القيود، الأذون، الحضور، الإجازات، الرواتب، المستخدمون) تُرفع تلقائيًا إلى Firebase Firestore وتُسحب عند فتح التطبيق على أي جهاز.

تأكد من ضبط قواعد Firestore للسماح بالقراءة والكتابة:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```
